Outline do HTML5:

Outline --> esboço, contorno, perfil, design...

Podemos usar a ferramenta HTML Outliner no chrome para ver!!!

Um outline em HTML é como o esqueleto dos tópicos principais de toda a página web, o título dos conteúdos, com suas sub-seções também. Confira usando o HTML Outline extencion. É importante pois na internet, há algoritmos que filtram nossas págins com base nos outlines delas, então se não for bem estruturada, podemos ter uma página má otimizada. Podemos validar nossas páginas em: Validator.nu

Todo site tem um outline básico, usando os cabeçalhos, e eles são importantes para estruturação dos conteúdos na página tanto para o site e acessibilidade quanto para os motores de busca.

Podemos ter vários resultados usando as mesmas estruturas de outlines, mas usando marcação correta, temos uma página mais significativa!!! Quero dizer assim: podemos criar um outline usando apenas cabeçalhos (h1 ao h6) e podemos criar um outline com os cabeçalhos e estruturando melhor ainda usando além os elementos article, aside, nav ou section para esta mesma página! Assim temos cabeçalhos aninhados para cada section por exemplo.

Para estruturar um outline, podemos usar elementos de cabeçalho, e o HTML5 tem quatro sectioning content: article, aside, nav e section. Eles demarcam seções distintas dentro de um documento e definem o escopo dos elementos H1 ao H6. Header e footer também vão ser usados para o outline da página!

Ou seja: o Outline vai definir a estrutura básca de níveis de conteúdos dentro da página. Então o conteúdo principal e geral poderia ser o H1, já outro que descende do título principal poderia ser um H2. Agora usando os sectioning contents, temos mais ferramentas para estruturar nosso outline da página. É como se fosse o menu de um produto e cada parte do manual tem seu título e seus sub-títulos comentando sobre algum assunto mais específico (h3 por exemplo) dentro do conteúdo principal (h1).

Não precisamos necessariamente (não é obrigatório) começar um sub-título com um H1 dentro de um section por exemplo, já que ele pode estar dentro de um outro H1 já existente. Mas é boa-prática seccionarmos o documento assim!

Uma hierarquia significativa de cabeçalho e outline faz com que a navegação seja mais fácil e melhora a experiência dos visitantes!!!
