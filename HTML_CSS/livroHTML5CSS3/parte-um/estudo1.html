O uso do HTML serve para estruturar conteúdos, definir seus significados (semântica) e é entendido por navegadores principalmente, por isso é padrão para páginas web.

Uma página em HTML possui 4 componentes básicos:

-> CONTEÚDO DE TEXTO

-> REFERÊNCIAS A OUTROS ARQUIVOS (links e arquivos)

-> MARCADORES (TAGS)

-> METADADOS (informações em html para o navegador, não é visto pelo usuário final. Podemos entender como Informações Instrucionais)



O conteúdo de TITLE é o único visível para o usuário na parte de metadados! O resto são informações para o navegador, sites de busca e intruções.



Um dos pontos importantes e primeiro passo a pensar, não importando como vão aparecer na tela de diversos dispositivos, é a acessibilidade a todos,
isso o HTML permite.
O HTML é universal também porque ele é apenas um arquivo de texto, podendo ser lido por vários programas e pelo navegador de diversos dispositivos.
por isso informamos no começo do arquivo texto que ele é do tipo HTML, ou seja !DOCTYPE html. este tipo informa ao navegador que se trata de um arquivo HTML de versão 5.

Uma prática bem conhecida é a de MELHORIA CONTÍNUA: Escrever uma página HTML que seja acessível a navegadores que apenas vão renderizar o HTML após bem estruturada, acrescentar a estilização em CSS e após acrescentar a programação Javascript!

A estrutura mínima de um documento HTML é: doctype,html,head,(metadados),/head,body,(conteúdo),/body,/html

O documento HTML e o fim da página termina com o fechamento da TAG /html e todo conteúdo dentro de body é o visível na tela do usuário

A paragrafação no código escrito pelo desenvolvedor não importa, mas é feito de forma a melhor entender o código (os aninhamentos), pois o que vai importar ao navegador são as instruções das TAGs e não como o conteúdo esta escrito. Lembrar que conteúdo vai ser diferente de valores em TAGs, os valores devem ser informados corretamente!

O sistema inteligente HTML é a semântica do conteúdo: ou seja, o significado. É como se fosse acrescentado as informações para aquele conteúdo e essas informações são chamadas marcações.

O HTML não define como o conteúdo deverá aparecer na tela, mas sim o que significa aquele conteúdo! Quem define o estilo é o CSS

ENTÃO POR QUE HÁ UMA ESTILIZAÇÃO MÍNIMA MESMO SEM TERMOS CRIADO UM CSS? Porque cada navegador possui um arquivo CSS embutido definindo padrões para as Tags, a menos que sobreescrevemos este estilo com outro arquivo CSS! Por isso cada navegador muda sutilmente seus estilos padrões, mas em geral são bem consistentes!
