LINK PARA HTML

O uso do atributo rel= em um link indica que tipo de link é, externo por exemplo, e o uso do atributo title= nos mostra um texto quando passamos o cursor por cima dele
há várias formas de usar ele, mas é bastante usado em links e suas tags, exemplos:
stylesheet: Indica que o recurso vinculado é uma folha de estilo CSS.
nofollow: Informa aos motores de busca que não devem seguir o link. Isso é frequentemente usado em links que não devem passar "link juice" (autoridade) para o site vinculado.
noopener: Usado em links que abrem em uma nova aba (target="_blank"). Melhora a segurança, evitando que a nova página tenha acesso à página de origem.
noreferrer: Similar ao noopener, mas também evita que a nova página receba informações sobre a página de origem.
prev: Indica que o link leva à página anterior em uma série de páginas.
next: Indica que o link leva à próxima página em uma série de páginas.
alternate: Usado para especificar uma versão alternativa do documento, como uma versão em outro idioma.
icon: Indica que o recurso vinculado é um ícone para o site (como um favicon).
help: Indica que o link leva a uma página de ajuda.
license: Indica que o link leva a informações sobre a licença do conteúdo.
dns-prefetch: Sugere que o navegador deve resolver o nome de domínio do recurso vinculado antes de ele ser realmente necessário.
O atributo rel é muito útil para definir a relação entre diferentes documentos e melhorar a navegação e a segurança. A escolha do valor adequado depende do contexto em que você está usando o link.

HTML: semântica do conteúdo
CSS: estilo do conteúdo
Javascript: Lógica de programação na página

No HTML 5, a separação de funções está mais definida, pois a função de estilo é especifica de CSS e HTML é apenas semântica. O que torna a renderização das TAGs em elementos inline ou inblock são as folhas de estilo padrões do proprio navegador, o que podemos mudar com CSS. Elementos de nivel de linha agora são entendidos como Phrasing Content em HTML 5

Em CSS: nivel de bloco= conteúdos que são renderizados em suas próprias linhas, com quebra de linha para aquela TAG
nivel de linha= conteúdos que são renderizados junto com a linha de texto, sem ter quebra de linha por exemplo.

esses termos foram usados para o HTML antes do HTML 5, pois anteriormente estilos também eram mais usados com o prório HTML, mas atualmente no HTML 5 a estilização é de responsabilidade do CSS e semântica apenas do HTML!

No HTML 5: nivel de linha = Phrasing Content
nive de bloco = Podemos clasificá-los como elementos estruturais, mas são essencialmente o mesmo que inblock, apenas devemos entender que HTML tem o objetivo de semântica e não de estilo, e esses elementos são usados principalmente no CSS!

Apesar do HTML 5 não utilizar inline e inblock, eles ainda servem para entendermos os comportamentos padrão dos elementos em HTML

indiferente do padrão de apresentação dos elementos, é buscado o que melhor descreve o significado de seus conteúdos, sem levar em conta a apresentação.

A tag article e a tag div: article tem a semântica de um conteúdo que pode ser lido e distribuido de forma independente, já a tag div é um conteiner genérico para conteúdos sem semântica especifica. as tags também ajudam os navegadores e motores de busca por exemplo, a exibirem de forma melhor os conteúdos e até na acessibilidade a usuários, por exemplo: com a tag article, por possuia a semântica de um conteúdo independente de outros, podem ser interpretados sozinhos, os motores de busca podem filtrar melhor os conteúdos, já se for div, não tem semântica e nõa será bem processada o conteúdo daquela página em uma busca por exemplo!
por isso o HTML é mais focado na semântica e o CSS é na estilização!

Outro exemplo: a tag /em é enfase em algo do texto e é um phrasing content, por padrão ele é renderizado em itálico, mas não precisa. O que importa é a semântica que diz que enfaiza algo do texto e podemos modificar com o CSS depois.
a tag /p quer dizer parágrafo e pode ter uma ou inúmeras sentenças nela, assim como em um texto impresso!

Toda página HTML deve ter título, usamos H1 - H6 (ajuda motores de busca também)
